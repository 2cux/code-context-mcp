# CodeContext MCP v0.3.0-rc Release Notes

> **Historical release candidate (2026-06-13).** This is not the current release. The current stable release is v1.0.0. See [Known Limitations](#known-limitations).

## Highlights

- **Safe Agent Mode**: Default `MCP_TOOL_MODE=agent` exposes only 7 audited tools. Dangerous tools (`delete_original`, `cleanup_originals`) and harness tools are hidden from AI agents.
- **`run_context_flow`**: Unified agent-facing entrypoint. Single tool call runs compress ΓåÆ remember ΓåÆ recall in one pass. 95% usability score in agent-mode evaluation.
- **Real MCP Adapter**: Shared `toolRegistry.ts` drives both the MCP server and the real adapter. No more stub adapter ΓÇö smoke tests exercise all 14 production tools via real handlers.
- **Tool Mode Security**: Three-tier tool surface (agent/dev/test) enforced at both ListTools and CallTool layers. Security regression test suite validates all 18 tools across all modes.
- **Performance Hardening**: Cache cold/warm split analysis, extreme memory guard with auto-sampling, schema version tracking to skip redundant migrations.
- **Clean Install Smoke**: Automated smoke test validates install ΓåÆ build ΓåÆ CLI ΓåÆ MCP server ΓåÆ npm pack in one command.

## Known Limitations

- **PERF_TEST_EXTREME** requires >16GB RAM for full execution. 8ΓÇô16GB machines use auto-sampling (20% slices).
- **Cache first-run timing**: New-process persistent hits include ~50ΓÇô90ms DB reopen overhead. Same-process warm hits are ~1ms.
- **CodeGraph index**: 3079 edges dropped during indexing ΓÇö impact analysis may be unreliable. Use `codegraph doctor --repair`.
- **No image/binary compression**: Out of scope for v1.
- **No cloud sync**: Local-first only.
