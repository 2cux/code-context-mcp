# CodeContext MCP v0.3.0-rc Release Notes

> ⚠️ **Release Candidate** — not a stable release. See [Known Limitations](#known-limitations).

## Highlights

- **Safe Agent Mode**: Default `MCP_TOOL_MODE=agent` exposes only 7 audited tools. Dangerous tools (`delete_original`, `cleanup_originals`) and harness tools are hidden from AI agents.
- **`run_context_flow`**: Unified agent-facing entrypoint. Single tool call runs compress → remember → recall in one pass. 95% usability score in agent-mode evaluation.
- **Real MCP Adapter**: Shared `toolRegistry.ts` drives both the MCP server and the real adapter. No more stub adapter — smoke tests exercise all 14 production tools via real handlers.
- **Tool Mode Security**: Three-tier tool surface (agent/dev/test) enforced at both ListTools and CallTool layers. Security regression test suite validates all 18 tools across all modes.
- **Performance Hardening**: Cache cold/warm split analysis, extreme memory guard with auto-sampling, schema version tracking to skip redundant migrations.
- **Clean Install Smoke**: Automated smoke test validates install → build → CLI → MCP server → npm pack in one command.

## Known Limitations

- **PERF_TEST_EXTREME** requires >16GB RAM for full execution. 8–16GB machines use auto-sampling (20% slices).
- **Cache first-run timing**: New-process persistent hits include ~50–90ms DB reopen overhead. Same-process warm hits are ~1ms.
- **CodeGraph index**: 3079 edges dropped during indexing — impact analysis may be unreliable. Use `codegraph doctor --repair`.
- **No image/binary compression**: Out of scope for v1.
- **No cloud sync**: Local-first only.

## MCP Configuration

### Agent Mode (recommended for AI coding agents)

```json
{
  "mcpServers": {
    "code-context": {
      "command": "node",
      "args": ["dist/index.js"],
      "env": {
        "MCP_TOOL_MODE": "agent"
      }
    }
  }
}
```

## Upgrade Notes

- Default mode is now `agent` (7 tools). Previously all tools were exposed.
- Set `MCP_TOOL_MODE=dev` to restore full 18-tool access.
- `run_context_flow` replaces manual compress → remember → recall chains.
- Cache warm performance should be interpreted with cold/warm split (see `reports/performance/cache-warm-analysis.md`).
